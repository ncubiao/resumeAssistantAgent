# 项目完整计划 (Project Plan)

> 本文档是 resumeAssistantAgent 的详细执行计划，供后续开发与优化参考。

## 1. 项目定位

- **项目类型**: 个人练手项目（Side Project）
- **核心价值**: 简历分析 AI Agent，服务于招聘方 & 求职方
- **开发原则**: 技术栈完整优先，效果可逐步打磨

## 2. 目标用户与核心场景

### 2.1 招聘方场景
- 上传简历 → 自动解析结构化
- 输入 JD → 计算单/多份简历匹配度
- 候选人横向对比与排序

### 2.2 求职方场景
- 上传自己的简历 → 获取结构化信息 + 亮点/不足
- 输入目标岗位 → 获取简历优化建议
- 按目标 JD 智能润色部分段落

## 3. 实施阶段（6 阶段）

| 阶段 | 名称 | 完成内容 | 估算工作量 |
|------|------|---------|-----------|
| 阶段 1 | 项目骨架搭建 | 目录结构、FastAPI 脚手架、Streamlit 基础页、Docker 配置、CI 框架 | 1-2 天 |
| 阶段 2 | 数据库 & 向量检索 | PostgreSQL 建表、SQLAlchemy ORM、FAISS 封装、简历 CRUD API | 2-3 天 |
| 阶段 3 | 简历解析模块 | PDF/Word 文本提取 + LLM 结构化提取 + Parser Agent + 缓存 | 2-3 天 |
| 阶段 4 | 核心 Agent 开发 | LangGraph 多 Agent 编排、匹配/优化 Agent、Prompt 模板 | 3-5 天 |
| 阶段 5 | 前端 UI 完善 | Streamlit 多页面、可视化图表、交互优化 | 2-3 天 |
| 阶段 6 | 工程化完善 | CI/CD、测试、日志、监控、文档 | 2-3 天 |

## 4. 里程碑（Milestones）

- **M1 v0.1**: 可以上传简历、完成基本解析
- **M2 v0.2**: 可以完成 JD-简历匹配评分
- **M3 v0.3**: 可以给出简历优化建议
- **M4 v0.4**: 多候选人对比 + 向量检索
- **M5 v1.0**: 工程化完善、可 Docker 一键部署

## 5. 技术决策记录

### 5.1 为什么选 LangGraph 而不是 AutoGen？
- LangGraph 更轻量，StateGraph 的思维模型更适合练手
- 与 FastAPI 集成更直观，便于控制

### 5.2 为什么选 Streamlit 而不是 React？
- 练手项目优先效果产出，Streamlit 无需写前端
- 后续可替换为 React，后端 API 保持不变

### 5.3 向量检索选 FAISS 而不是 Milvus？
- FAISS 纯本地，无需额外服务，Docker 启动更简单
- 项目成熟后可切换为 Milvus / Qdrant

## 6. 质量保障

- 每个模块必须有单元测试
- PR 必须通过 CI（ruff + mypy + pytest）
- 核心逻辑必须写日志

## 7. 后续可扩展方向

- 简历 OCR（图片简历）
- 多语言支持（英文简历）
- 面试问题自动生成
- 候选人漏斗分析
- RAG 知识库（公司 JD 模板、行业岗位标准）
